# Divan v0.15.0 Project OS Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn Divan's repository-only standards into a portable, supervised Project OS that carries an intent through specification, implementation evidence, preview, release, live readback, and applicable SEO checks.

**Architecture:** Keep the stdlib Python Company OS as the deterministic policy kernel and preserve the existing five packages and 41 skills. Extend the kernel with Unicode routing, recursive manifest-based project discovery, multi-workflow composition, project bootstrap/state modules, provider capability contracts, project standards, and portable CI assets. Machine-facing names remain English; Turkish and English reader-facing copy remain first-class.

**Tech Stack:** Python 3.11+ standard library, JSON registries, unittest, GitHub composite actions, GitHub Pages/Wiki, optional pinned Lighthouse CI and Lychee tools.

## Global Constraints

- Work only in the isolated `codex/v015-project-os` worktree created from `origin/main`; never modify the user's dirty local `main`.
- Use TDD for every production behavior: add one focused failing test, observe the expected failure, implement the minimum behavior, and observe the test pass.
- Preserve the existing five marketplace packages and 41 discoverable skills.
- Canonical machine interfaces, filenames, identifiers, JSON keys, and CLI flags are English; reader-facing output supports correct UTF-8 English and Turkish.
- Existing Turkish entrypoints remain narrow deprecated wrappers throughout v0.x and contain no duplicate implementation.
- Mutating project commands are dry-run by default and require `--execute`.
- Never execute detected project code during inspection; bound file size, traversal depth, workspace count, and symlink behavior.
- Every changed path is classified or explicitly reported as `unclassified`; no silent impact gaps.
- Never write secrets, personal absolute paths, hidden reasoning, tokens, or unrelated plugin identities to public receipts.
- Provider absence is `BLOCKED` with an actionable missing capability, never synthetic success.
- Upstream code is not copied or forked without a verified license, exact commit pin, and concrete product need.
- v1 independent-adoption readiness stays 7/8 until a non-owner supplies reproducible evidence.
