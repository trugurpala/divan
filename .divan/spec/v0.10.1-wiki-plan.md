# v0.10.1 Wiki Yayın Planı

## Hedef

GitHub Wiki'nin varsayılan karşılama metninde kalmasını engelle; Wiki'yi
vibe coder'ın niyetinden güncel rehbere giden, repoda sürümlenen ve `main`
sonrası canlılığı doğrulanan bir ürün yüzeyi yap.

## Kabul ölçütleri

1. `Home` sürümü, ürün vaadini ve beş temel niyeti açıkça gösterir.
2. Kurulum, değer, katalog, test, standart, durum/roadmap ve kaldırma sayfaları
   manifestten deterministik üretilir.
3. OpenAI Agents SDK ile portable Agent Skills aynı şeymiş gibi sunulmaz.
4. PR kaynağı derler; `main` Wiki Git deposuna yazar ve canlı `Home.md`yi okur.
5. Wiki yayını başarısızsa iş akışı kırılır; teslim “canlı güncel” sayılmaz.
6. Sadrazam bu kuralı sonraki projelerde de otomatik uygular.
