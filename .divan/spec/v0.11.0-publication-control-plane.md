# v0.11.0 Yayın Kontrol Düzlemi Planı

Tarih: 2026-07-18

## Kullanıcı sorunu

Ürün kodu değişirken README, kitap/Wiki, site, yol haritası veya GitHub Release
sayfasının eski kalması; kullanıcının her oturumda aynı hatırlatmayı yapmasına
neden oluyor. Sohbet hafızası bu sorunu güvenilir biçimde çözmez.

## Karar

`VERSION` tek sürüm kaynağıdır. `release-manifest.json` hangi kamusal yüzeylerin
bu sürümü taşıması gerektiğini bildirir. `/yayin` ve `scripts/yayin.py` yeni
sürümün deterministik alanlarını hazırlar; CI sapmayı kırar. CHANGELOG ve
BLUEPRINT anlatısı otomatik uydurulmaz, yapılan gerçek işle birlikte yazılır.

`main` sonrası yayın workflow'u önce Pages ve Wiki'nin aynı sürümü gösterdiğini
kanıtlar, sonra CHANGELOG bölümünden tag/GitHub Release üretir. Var olan etiket
başka commit'e taşınmaz. v1 durumu ayrı makine-okunur kabul defterinden üretilir.

## Kabul ölçütleri

1. Tek komut README/TR+EN, Wiki kaynakları, Pages/site ve marketplace sürümünü hazırlar.
2. Eksik/eskimiş yüzey, changelog veya v1 karnesi unit test/CI'ı kırar.
3. GitHub Release yalnız yerel/resmî teftiş ve canlı Pages+Wiki eşliğinden sonra oluşur.
4. Release notu mevcut sürümün CHANGELOG bölümünden gelir; yeniden koşu etiketi taşımaz.
5. Claude Code şeması ve Codex Linux/macOS/Windows kur-kaldır yolu temiz host matrisinde sınanır.
6. Kurulum kaydı dışındaki hedefler kaldırma betiği tarafından silinemez.
7. Bağımsız kabul formu gerçek environment, sabit release, komut, görev ve kanıt ister.
8. Gerçek ajan/hakem ve bağımsız kullanıcı kanıtı gelmeden v1 iddiası yapılmaz.

## Kaynak kararları

- GitHub Docs: Release için job düzeyinde `contents: write`; diğer yetkiler
  verilmez. `GITHUB_TOKEN` kaynaklı olayların recursion oluşturmaması ve
  workflow/ref bazlı concurrency kullanılır.
- Codex/Agent Skills: taşınabilir skill katmanı ile Claude Code'a özgü komut,
  hook ve subagent katmanı birbirine karıştırılmaz.

## Kapsam dışı

- Kanıtsız performans, gelir veya “dünya standardı” iddiası.
- API anahtarı talep eden gerçek-agent koşusunu kullanıcı izni olmadan başlatmak.
- Release workflow'unda kod/README'yi gizlice commit edip ana dala yazmak.
