# v0.10.3 Kendini Tanıma ve Wiki Teftiş Planı

## Hedef

Divan'ın “ne yaparız?” sorusunu kaynak gerçekliğinden üret; GitHub, Wiki,
OpenAI/Codex, Context7 ve Mühürdar yüzeylerini aynı teftişte sınayıp kullanıcıya
bozuk veya doğrulanmamış vitrin sunma.

## Kabul ölçütleri

1. 41 skill'in adı ve açıklaması doğrudan frontmatter'dan deterministik üretilir.
2. Çok satırlı YAML açıklaması katalogda bozulursa unit test ve CI kırılır.
3. Wiki kaynağı 16 çıktıya derlenir; canlı depo yoksa hata tek gerekli kullanıcı
   eylemini (`Save Page`) açıkça söyler.
4. GitHub Actions resmî depolardaki güncel Node 24 tabanlı major sürümleri kullanır.
5. Context7 GitHub Docs, güncel Codex kılavuzu ve GitHub workflow kanıtı ayrı
   kaynaklar olarak kayda geçer.
6. Etkileşimli harita 41 skill'i beş pakette filtreler ve her skill'in görevini
   seçildiğinde açıklar.
7. Yerel test, resmî doğrulayıcılar, PR CI, `main` ve canlı yüzey ayrı doğrulanır.

## Dış kapı

GitHub Wiki Git deposu yalnız ilk Wiki sayfası web arayüzünde bir kez
kaydedildikten sonra oluşur. Connector'da Wiki sayfası oluşturma aracı yoktur;
bu tek eylem kullanıcı hesabının GitHub arayüzünde tamamlanmalıdır.
